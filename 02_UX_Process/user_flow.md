# User Flow

## Primary Booking Flow

User opens app
→ Selects service
→ Chooses barber
→ Selects available time slot
→ Verifies phone number through OTP
→ Reviews appointment details
→ Confirms booking
→ Receives booking confirmation

## User Flow Cases

### Case 1 — Returning Customer

A returning customer who already knows their preferred barber:

Select Service
→ Select Known Barber
→ Choose Time Slot
→ Confirm Booking

### Case 2 — First-Time Customer

A first-time customer unfamiliar with barbers:

Select Service
→ Recommended Barber appears
→ Select Time Slot
→ Confirm Booking

## Rescheduling Flow

Appointments
→ Upcoming Booking
→ Reschedule
→ Select New Time
→ Confirm Update

## Completed Booking Flow

Appointments
→ Completed Tab
→ Book Again
→ Previous preferences prefilled
